type MOO = ReaderT Environment (ContT TaskDisposition (StateT TaskState STM))Source

This is the basic MOO monad transformer stack. A computation of type MOO a is an STM transaction that returns a value of type a within an environment that supports state, continuations, and local modification.

newTask :: TVar World -> ObjId -> MOO Value -> IO TaskSource

runTask :: Task -> IO (Maybe Value)Source

Run a task in a new Haskell thread, returning either the value produced by the task, or Nothing if the task suspends or aborts before producing a value. If the task suspends, it may continue running after this function returns. After the task is finished, it is removed from the task queue.

forkTask :: TaskId -> Integer -> MOO Value -> MOO ()Source

Create and queue a task to run the given computation after the given microsecond delay. E_INVARG may be raised if the delay is out of acceptable range. (The given TaskId should have been reserved by a call to newTaskId.)

interrupt :: TaskDisposition -> MOO aSource

Commit the current task's transaction, and return to the task dispatcher with an interrupt request. The task dispatcher may resume execution of the task later if the request is one which supplies an appropriate continuation.

requestIO :: IO a -> MOO aSource

Interrupt the current task to perform the given IO computation, and return the result. Note this implies a commit of the task's STM transaction.

liftSTM :: STM a -> MOO aSource

Lift an STM transaction into the MOO monad.

delayIO :: IO () -> MOO ()Source

Perform the given IO computation after the current task commits its STM transaction.

Since IO can't be performed within a transaction, this is a simple alternative when the value returned by the IO isn't needed.
